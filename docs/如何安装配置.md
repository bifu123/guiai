# Ylbot GUI Agent 安装与配置指南

本文档将指导您如何从零开始安装、配置并运行 Ylbot GUI Agent。

## 1. 环境准备

### 1.1 操作系统要求
- **服务端 (Agent 大脑)**: Windows, Linux, 或 macOS 均可。
- **客户端 (执行器)**: 
  - PC 端: Windows 10/11 (推荐), macOS, Linux (需支持 GUI)。
  - 移动端: Android 手机 (需开启 USB 调试或无线调试)。

### 1.2 软件依赖
- **Python**: 3.10 或更高版本。
- **Redis**: 必须安装并运行 Redis 服务（用于状态管理和防并发锁）。
  - Windows 用户可下载 [Memurai](https://github.com/memurai/memurai-developer) 或使用 WSL 安装 Redis。
  - Linux/macOS 用户直接通过包管理器安装即可 (`apt install redis` 或 `brew install redis`)。

---

## 2. 安装步骤

### 2.1 克隆项目
```bash
git clone https://github.com/bifu123/guiai.git
cd guiai
```

### 2.2 创建虚拟环境 (推荐)
```bash
python -m venv venv
# Windows 激活:
venv\Scripts\activate
# Linux/macOS 激活:
source venv/bin/activate
```

### 2.3 安装依赖包
```bash
pip install -r requirements.txt
```

## 3. 环境变量配置

项目使用 `.env` 文件来管理所有敏感的 API Key 和服务地址。

1. 在项目根目录下，找到 `.env.example` 文件。
2. 将其复制并重命名为 `.env`。
3. 打开 `.env` 文件，根据您的实际情况填入配置：

### 3.1 文本模型 (LLM) 配置
用于将自然语言指令拆解为具体的 GUI 动作。
```env
LLM_API_KEY=your_api_key_here
LLM_MODEL=deepseek-v4-flash
LLM_BASE_URL=https://api.deepseek.com
```
*推荐使用 DeepSeek，性价比极高。*

### 3.2 视觉模型 (VLM) 配置
用于验证任务是否成功，或理解复杂的屏幕内容。
```env
# 推荐使用本地部署的 Ollama Qwen3-VL
OLLAMA_HOST=http://127.0.0.1:11434
```
*如果您没有本地算力，也可以配置 `ZHIPU_API_KEY` 或 `SILICONFLOW_API_KEY` 使用云端模型。*

### 3.3 OCR 模型配置
用于在屏幕截图中精准定位目标文字的物理坐标。
```env
# 推荐使用百度 OCR (免费额度高，精准度极高)
BAIDU_OCR_CLIENT_ID=your_baidu_client_id
BAIDU_OCR_CLIENT_SECRET=your_baidu_client_secret
```
*您也可以配置 `QWEN_OCR_API_KEY` 使用阿里千问 OCR，或者直接复用上面的 Ollama Qwen3-VL。*

### 3.4 服务端与客户端地址配置
```env
# GUI 服务端地址 (API 网关)
GUI_SERVER_URL=http://127.0.0.1:8001/api/run_for_agent

# GUI 客户端地址 (PC 执行器)
GUI_CLIENT_URL=http://127.0.0.1:8000/execute
```

### 3.5 Redis 配置
用于状态管理、防并发锁和历史轨迹存储。
```env
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_DB=13
```

> **详细的模型切换方法，请参考：[怎样更换视觉模型和OCR模型.md](../怎样更换视觉模型和OCR模型.md)**

## 4. 启动服务

为了让 Agent 正常工作，您需要启动以下几个组件：

### 4.1 启动 Redis
确保您的 Redis 服务正在运行（默认端口 6379）。

### 4.2 启动客户端 (执行器)
在需要被控制的电脑上运行：
```bash
python gui_client.py
```
*默认会在 `8000` 端口启动 HTTP 服务。*

### 4.3 启动服务端 (API 网关)
在部署 Agent 大脑的服务器上运行：
```bash
python gui_server.py
```
*默认会在 `8001` 端口启动 HTTP 服务。*

## 5. 测试运行

服务启动后，您可以通过发送 HTTP 请求来测试 Agent 是否正常工作。

### 5.1 测试基础动作 (直接调用 Client)
打开一个新的终端，发送以下 curl 命令测试鼠标点击：
```bash
curl -X POST http://127.0.0.1:8000/execute \
     -H "Content-Type: application/json" \
     -d '{"action": "click", "coords": [500, 500]}'
```
*如果目标电脑的鼠标移动到了 (500, 500) 并点击，说明客户端正常。*

### 5.2 测试自然语言意图 (调用 Server)
发送自然语言指令给 Agent 大脑：
```bash
curl -X POST http://127.0.0.1:8001/api/run_for_agent \
     -H "Content-Type: application/json" \
     -d '{
           "user_id": "test_user_01",
           "intent": "在桌面上找到此电脑图标并双击打开它"
         }'
```
*Agent 将会接管任务，调用大模型进行思考，定位图标，并最终控制客户端执行双击操作。*

## 6. 常见问题 (FAQ)

**Q: 运行 `gui_client.py` 时提示缺少 `pyautogui` 依赖？**
A: Linux 用户可能需要安装额外的系统依赖：`sudo apt-get install python3-tk python3-dev`。

**Q: Agent 提示 "OCR 定位失败"？**
A: 请检查您的 `.env` 文件中是否正确配置了 OCR 的 API Key，并确保网络畅通。

**Q: 如何录制固定的 RPA 流程？**
A: 请参考 [录制说明.md](../录制说明.md) 使用 `gui_recorder.py` 工具。
