# GUI 自动化录制与回放说明文档

本文档介绍了如何使用本项目的录制工具 (`gui_recorder.py`) 记录鼠标和键盘操作，并使用执行工具 (`gui_tools.py`) 进行自动化回放，同时支持动态参数注入。

---

## 1. 录制操作轨迹 (gui_recorder.py)

`gui_recorder.py` 是一个全局监听工具，可以记录你在屏幕上的鼠标点击、双击、拖拽以及键盘输入，并将其保存为 JSON 格式的流程文件。

### 1.1 启动录制
在命令行中运行以下命令启动录制器：
```bash
python gui_recorder.py
```

### 1.2 快捷键说明
启动后，程序会在后台监听以下快捷键。将鼠标移动到目标位置后，按下对应的快捷键即可记录动作：

- **[F8] 录制单击**：记录当前鼠标位置的左键单击。
- **[F9] 录制双击**：记录当前鼠标位置的左键双击。
- **[F10] 录制键盘输入**：记录文本输入操作（坐标默认为 [0,0]，通常在点击输入框后使用）。
- **[F11] 录制滚动**：记录当前鼠标位置的滚轮滚动。
- **[F12] 录制拖拽**：需要按两次。第一次在起点按下记录起点，移动鼠标到终点后第二次按下记录终点。
- **[Esc] 结束录制并保存**：停止监听并进入保存流程。

### 1.3 弹窗交互
为了避免控制台焦点丢失的问题，每次按下快捷键记录动作后，屏幕中央会弹出一个**置顶的输入框**：
1. **输入文本**（仅 F10）：提示你输入需要让机器自动打字的文本内容。
2. **输入注释**：提示你为当前步骤添加描述（例如：“点击登录按钮”）。直接回车会使用默认描述。

### 1.4 保存录制
按下 `Esc` 键后，会弹窗提示输入要保存的 JSON 文件名（默认 `record_flow.json`）。
- 如果输入的文件名没有 `.json` 后缀，程序会自动补全。
- 保存成功后，控制台会打印出该文件的**绝对路径**。

## 2. 回放操作轨迹 (gui_tools.py)

录制生成的 JSON 文件可以通过 `gui_tools.py` 中的 `execute_manual_flow` 函数进行回放。

### 2.1 基本回放测试
你可以直接运行 `gui_tools.py` 进行测试：
```bash
python gui_tools.py
```
选择 `2` (测试手动流程自动化)，然后选择 `B` (读取 JSON 文件)，输入你刚才录制的文件路径即可开始回放。

### 2.2 在代码中调用
在你的 Python 代码中，可以这样调用：
```python
from gui_tools import execute_manual_flow

# 直接传入 JSON 文件路径
result = execute_manual_flow("record_flow.json")

# 检查执行结果
if result["status"] == "success":
    print("执行成功！")
```

### 2.3 调整执行速度
有些程序加载较慢，你可以通过 `time_sleep` 参数调整每一步操作之间的等待时间（默认 3.0 秒）：
```python
# 每步操作后等待 5 秒
execute_manual_flow("record_flow.json", time_sleep=5.0)
```

## 3. 动态参数支持 (高级用法)

在实际业务中，像“用户名”、“密码”这样的输入内容通常是动态的（例如从数据库读取）。本工具支持通过**占位符**的方式实现动态参数注入。

### 3.1 修改 JSON 文件使用占位符
打开录制好的 JSON 文件，找到需要动态输入的步骤，将 `text` 字段的值修改为 `${变量名}` 的格式。

**示例：**
```json
{
    "auto": {
        "action": "type",
        "coords": [0, 0],
        "text": "${username}",  // 这里使用了占位符
        "key": ""
    },
    "description": "输入用户名"
}
```

### 3.2 传入动态参数字典
在调用 `execute_manual_flow` 时，通过 `params` 参数传入一个字典，字典的键名需要与占位符中的变量名一致。

**示例代码：**
```python
from gui_tools import execute_manual_flow

# 从数据库或其他地方获取的真实数据
user_data = {
    "username": "root",
    "password": "my_secret_password"
}

# 执行时传入 params
result = execute_manual_flow(
    "启动vspher.json", 
    params=user_data
)
```
执行时，程序会自动将 `${username}` 替换为 `root`，将 `${password}` 替换为 `my_secret_password` 并进行输入。

## 4. JSON 轨迹文件格式参考

一个标准的轨迹文件结构如下：
```json
[
    {
        "auto": {
            "action": "double_click",
            "coords": [173, 1065],
            "text": "",
            "key": ""
        },
        "description": "打开vsphere"
    },
    {
        "auto": {
            "action": "type",
            "coords": [0, 0],
            "text": "${username}",
            "key": ""
        },
        "description": "输入用户名"
    }
]
```
- `action`: 动作类型 (`click`, `double_click`, `type`, `scroll`, `drag`)
- `coords`: 坐标 `[x, y]`
- `text`: 输入的文本或滚动方向、拖拽终点坐标
- `key`: 特殊按键（暂留）
- `description`: 步骤的中文描述，用于日志打印
